Questions & Answers.

Everything you need to know about MLCast, weather nowcasting, and how to get involved in the community.

What is MLCast?

expand_more

MLCast is an open-source community developing machine learning models for weather nowcasting. The project aims to make advanced weather prediction accessible globally through collaborative development.

Why weather nowcasting?

expand_more

Weather nowcasting focuses on predicting weather conditions over the next few minutes to several hours. These predictions are vital for tracking severe weather events and complement traditional forecasting approaches.

How do I get started?

expand_more

Begin by reviewing project documentation and repositories, then follow installation guides and tutorials. Contributors should explore beginner-friendly issues and read contribution guidelines.

How can I contribute?

expand_more

Contributions include developing features, training models, improving documentation, testing, contributing datasets, and participating in community discussions. All experience levels are welcome.

Do I need a GPU?

expand_more

GPUs aren't required for many contributions like documentation, testing, and data work. While training large machine learning models typically requires GPU resources, community members with GPU access can train models for shared use.

How do I train a model?

expand_more

The workflow involves installing dependencies, downloading datasets, validating data, configuring parameters, executing training scripts, and evaluating results through detailed model-specific guides.

Where does the data come from?

expand_more

Data originates from verified weather radar scans from five European countries currently, with satellite imagery forthcoming. All datasets undergo conversion to zarr format and validation.

How are models evaluated?

expand_more

Models use separate validation and test datasets with standard machine learning and meteorological metrics ensuring fair comparisons and reliable generalization.

How do I use a pretrained model?

expand_more

Download pretrained models from the repository, install required packages, load via provided APIs, and run inference with example notebooks available.

Where can I ask for help?

expand_more

Consult documentation, GitHub Issues/Discussions, or community channels including Slack and monthly Microsoft Teams meetings for support.
