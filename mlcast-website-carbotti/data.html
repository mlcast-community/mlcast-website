Option A: Direct GitHub Access

Use the most recent version of the catalog directly from our source repository.

pip install intake intake-xarray zarr jinja2

import intake
cat = intake.open_catalog("https://raw.githubusercontent.com/mlcast-community/mlcast-datasets/main/src/mlcast_datasets/catalog/catalog.yml")

Option B: Python Package

Install our stable mlcast-datasets package from PyPI for consistent versioning.

pip install mlcast-datasets

import mlcast_datasets
cat = mlcast_datasets.open_catalog()

Quick Exploration

Once you have the catalog open, you can start exploring the data instantly:

# List available sources
print(list(cat))
# -> ['precipitation']

# Access a specific radar dataset
ds = cat.precipitation.radklim_5_minutes.to_dask()

cloud_upload ARCO Standard

We adopt the ARCO (Analysis-Ready, Cloud-Optimized) pattern. Our data is stored in Zarr format, enabling partial reads and streaming directly from the cloud without massive local storage.

storage EWC Infrastructure

Hosted on the European Weather Cloud, our datasets benefit from high-throughput connectivity to meteorological supercomputers, minimizing egress latency.

code Code-Addressable

Data is not a static file; it's an object in your Python pipeline. Using intake, every dataset is queryable, version-controlled, and ready for training.

The Catalog Architecture

The MLCast Intake Catalog serves as the single source of truth for all community-curated data. By centralizing metadata and access paths, we ensure that every researcher uses the exact same data versioning, coordinate systems, and schema, eliminating the "data cleaning" bottleneck in machine learning experiments.

Transparency & Metadata

Every dataset in our catalog includes rich provenance metadata (Author, Conventions, Creation Date), ensuring full scientific traceability:

# Full dataset attributes available instantly
print(ds.attrs['reference'])
# -> '10.5676/DWD/RADKLIM_YW_V2017.002'

print(ds.attrs['mlcast_dataset_identifier'])
# -> 'DE-DWD-radar_precipitation-RADKLIM'
